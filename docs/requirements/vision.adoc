== Positionierung

=== Fachliche Motivation
//Erläutern Sie kurz den Hintergrund, in dem das Projekt angesiedelt ist. Welches Problem soll gelöst werden, wie ist es entstanden? Welche Verbesserung wird angestrebt. Achten Sie darauf, eine fachliche (organisatorische, betriebswirtschaftliche) Perspektive einzunehmen.

Viele junge Erwachsene – insbesondere Studierende und Berufstätige – möchten ihre Alltagsroutinen verbessern, etwa durch regelmäßige Bewegung, strukturierte Tagesplanung oder gesündere Ernährung. Während die Zielsetzung meist klar ist, fehlt es an nachhaltiger Motivation, positiver Verstärkung und emotionaler Bindung zur Gewohnheitsbildung.

Bestehende Habit-Tracker setzen meist auf Listen oder reine Zahlen, wodurch die emotionale Beteiligung gering bleibt und Nutzer:innen schnell das Interesse verlieren.

{project-system-name} begegnet diesem Problem mit einer spielerischen, emotional ansprechenden Komponente: einem digitalen Begleiter in Form eines Maskottchens (Tier), das die Nutzer:innen auf ihrer Gewohnheitsreise begleitet.

Der Einsatz eines solchen Begleiters schafft emotionale Bindung, stärkt das Verantwortungsgefühl und macht den Prozess der Verhaltensänderung positiv. Die Kombination aus Fortschritten und Gamification erhöht somit die langfristige Nutzungsdauer und Zielerreichung.

=== Zusätzliche Produktanforderungen

|===
| Anforderung | Priorität | Beschreibung | Geplantes Release

| Einfache Bedienbarkeit auf mobilen Endgeräten 
| Hoch 
| Das System muss vollständig responsive sein und auf Smartphones (iOS, Android) sowie Tablets intuitiv bedienbar bleiben. Layout und Interaktion sollen sich automatisch an Displaygröße und Eingabemethode (Touch) anpassen.
| 

| DSGVO-konforme Datenspeicherung
| Hoch
| Alle personenbezogenen Daten (z. B. E-Mail, Fortschritte, Nutzungsverhalten) müssen gemäß der Datenschutz-Grundverordnung verarbeitet und gespeichert werden. Nutzer:innen müssen aktiv in die Datennutzung einwilligen können.
| 

| Systemfeedback (Usability)
| Hoch
| Das System soll durchgehend Rückmeldung über Nutzeraktionen geben – etwa durch visuelle Bestätigung, Ladeindikatoren oder Erfolgsmeldungen. Außerdem wird ein kontinuierlicher Usability-Testprozess etabliert, um die Nutzerfreundlichkeit sicherzustellen.
| 

| Plattformunabhängigkeit
| Mittel
| Das System soll als Progressive Web App (PWA) konzipiert sein, um plattformübergreifend (Browser, Android, iOS) verfügbar zu sein, ohne separate native Apps zu erfordern.
| 

| Performance und Skalierbarkeit
| Mittel
| Ladezeiten sollen unter drei Sekunden bleiben, selbst bei steigender Nutzerzahl. Das Backend (z. B. Supabase) muss so ausgelegt sein, dass zukünftige Skalierung ohne Funktionsverlust möglich ist.
| 

| Offline-Funktionalität
| Mittel
| Nutzer:innen sollen ihre Gewohnheiten auch ohne Internetverbindung erfassen können; Daten werden bei der nächsten Verbindung automatisch synchronisiert.
| 

| Barrierefreiheit
| Mittel
| Das System soll grundlegende WCAG-Richtlinien erfüllen (z. B. ausreichende Kontraste, Screenreader-Kompatibilität, große Klickflächen).
| 

| Wartbarkeit und Erweiterbarkeit
| Niedrig
| Der Quellcode soll modular aufgebaut und dokumentiert werden, um zukünftige Feature-Erweiterungen (z. B. zusätzliche Maskottchen oder Social-Features) zu erleichtern.
| 

| Dokumentation
| Niedrig
| Es soll eine kurze Onlinehilfe (Onboarding-Tutorial + FAQ) bereitgestellt werden, die den Einstieg für neue Nutzer:innen erleichtert. Für Entwickler:innen wird eine interne technische Dokumentation gepflegt.
| 
|===
